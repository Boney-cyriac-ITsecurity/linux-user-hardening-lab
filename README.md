# Linux User Hardening Lab

## Objective
This lab demonstrates basic Linux user management and system hardening techniques.

The goal is to understand how improper user configuration can create security risks.

## Lab Scenario
Simulating a small server environment where:
- Multiple users exist
- Permissions are misconfigured
- Hardening steps are required

## Environment
- Virtual Machine
- Ubuntu Server

## Tasks Performed
- Created multiple users
- Configured groups
- Modified file permissions
- Restricted sudo access
- Verified access controls

## Security Focus
Understanding least privilege and proper user access control.
